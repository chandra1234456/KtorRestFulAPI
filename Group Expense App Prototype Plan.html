4. Key Backend Endpoints

  • POST /auth/login – User login, returns JWT
  • POST /auth/signup – Create new user
  • GET /groups – Get user groups
  • POST /groups – Create new group
  • GET /groups/{groupId}/expenses – Get expenses in a group
  • POST /groups/{groupId}/expenses – Add expense
  • GET /groups/{groupId}/balances – Get calculated balances

5. Key Android Screens

  • Login Screen – User login, JWT stored in SharedPreferences
  • Group List Screen – Shows all user groups
  • Add Expense Screen – Enter expense amount, payer, participants
  • Balance Screen – Displays net balance per user, +ve = receive, -ve = owes

6. Core Logic – Balance Calculation


fun calculateBalances(expenses: List): Map {
    val balances = mutableMapOf()

    expenses.forEach { expense ->
        val share = expense.amount / expense.participants.size

        expense.participants.forEach { userId ->
            balances[userId] = balances.getOrDefault(userId, 0.0) - share
        }

        balances[expense.paidBy] =
            balances.getOrDefault(expense.paidBy, 0.0) + expense.amount
    }
    return balances
}

7. Authentication & Security

  • JWT authentication in Ktor
  • Android sends JWT in headers for every API call
  • Backend verifies JWT and ensures only group members access group data

8. Local Testing Setup

  • Use H2 database for backend local testing

  • Configure Android app Ktor client to point to local backend
  • Test full flow: create user → create group → add expense → view balance
